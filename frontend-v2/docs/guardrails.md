# Frontend V2 Guardrails

## Allowed now

- Contract documents
- Scope boundaries
- Domain type drafts
- Public frontend env example
- Cleanup and packaging policies

## Not allowed yet

- UI screens
- Design system components
- API client implementation
- Dashboard
- Create Flow
- Studio
- Render Observatory
- Fake production features

## Reason

The project must remain backend-driven. Premature UI generation would recreate the same architectural debt that Phase 1 and Phase 2 are designed to eliminate.
